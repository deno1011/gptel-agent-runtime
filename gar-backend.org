#+TITLE: gar-backend: Ollama runtime utilities + backend registry forward decls
#+PROPERTY: header-args:emacs-lisp :tangle gar-backend.el :lexical t

Module of [[file:gptel-agent-runtime.org][gptel-agent-runtime]]. Provides the provider-neutral runtime layer
that mediates between gptel itself and the host's backend setup. The
backend constructors (=gptel-make-anthropic=, =gptel-make-openai=,
=gptel-make-ollama= ...) live in the consuming setup file, not here -- see
[[file:gptel-agent-runtime.org][the master]] for the architecture comment that catalogues each setup
responsibility.

What this module owns:

- forward =defvar= for =gptel-agent-runtime-backends= (alist of registered backends
  populated by the setup file) and =gptel-agent-runtime-ollama-backend= (the live
  Ollama backend object, when present)
- Ollama runtime utilities: =gptel-agent-runtime-ollama-running-p=,
  =gptel-agent-runtime-start-ollama-if-needed=,
  =gptel-agent-runtime-active-ollama-model=,
  =gptel-agent-runtime-use-default-local-model=, and the private URL
  helper =gptel-agent-runtime--ollama-url=
- model-id normalization helpers: =gptel-agent-runtime-model-id= (returns the
  symbol id from gptel metadata in any shape -- symbol, cons, vector)
  and =gptel-agent-runtime-backend-model-symbols= (lists a backend's model ids
  uniformly as symbols)
- =gptel-agent-runtime-register-model= -- adds (or replaces) one entry in the
  =gptel-agent-runtime-backends= alist; called by the user's backend setup and by
  the model-router

This module is provider-neutral: it neither requires nor instantiates
any specific gptel provider module. The setup file is responsible for
requiring provider modules and calling the constructors.

See also:
- [[file:gptel-agent-runtime.org][gptel-agent-runtime]] (master entrypoint that requires this module)
- [[file:gar-directives.org][gar-directives]] (=gptel-agent-runtime-sync-directive-for-current-runtime= is
  called from =gptel-agent-runtime-use-default-local-model= when the
  active Ollama model is selected; gar-directives loads before
  gar-backend in the master's =(require)= order so the symbol is
  bound by the time the function is invoked)

* Backend registry forward declarations

These are populated by the consuming setup file. Forward-declaring them
here means the runtime layer (model router, mission control,
=gptel-agent-runtime-use-default-local-model=) can reference them safely
even when the setup file has not yet run.

#+begin_src emacs-lisp
(defvar gptel-agent-runtime-backends nil
  "Alist of (DISPLAY-NAME . (BACKEND . MODEL)) for all backends.
Populated by the consuming setup file; the runtime package only forward-
declares it.")

(defvar gptel-agent-runtime-ollama-backend nil
  "The registered Ollama backend, when the setup file has installed one.
Forward-declared by the package; populated by the setup file.")

;; Backwards-compat aliases for the legacy `my/gptel-*' names. These are
;; storage cells: user setup files commonly do `(setq my/gptel-backends
;; ...)' AFTER `(require 'gptel-agent-runtime)' to register their
;; provider backends. `defvaralias' makes both names refer to the same
;; underlying cell, so user `setq's land where the new
;; `gptel-agent-runtime-backends' reader looks. Installing the alias
;; BEFORE the setup file's `setq' (which is true because we are inside
;; the package, loaded by `(require ...)' first) is what makes this
;; work.
(defvaralias 'my/gptel-backends 'gptel-agent-runtime-backends)
(defvaralias 'my/gptel-ollama-backend 'gptel-agent-runtime-ollama-backend)
#+end_src

* Ollama runtime utilities

Synchronous probes against the local Ollama server, plus a default
launcher (=start-ollama-if-needed=) and an active-model detector. The
=use-default-local-model= function is the user-facing entry: it prefers
the Ollama model the user has loaded right now and falls back to the
configured =gptel-agent-runtime-default-local-model=.

#+begin_src emacs-lisp
(defun gptel-agent-runtime--ollama-url (&optional path)
  "Return the local Ollama URL for PATH."
  (format "http://%s%s"
          gptel-agent-runtime-ollama-host
          (or path "")))

(defun gptel-agent-runtime-ollama-running-p ()
  "Return non-nil when the configured Ollama server responds."
  (condition-case nil
      (let ((buf (url-retrieve-synchronously
                  (gptel-agent-runtime--ollama-url "/api/tags") t t 1)))
        (when buf
          (kill-buffer buf)
          t))
    (error nil)))

(defun gptel-agent-runtime-start-ollama-if-needed ()
  "Start Ollama in the background if configured and not already running."
  (when (and gptel-agent-runtime-auto-start-ollama
             (executable-find gptel-agent-runtime-ollama-command)
             (not (gptel-agent-runtime-ollama-running-p)))
    (let ((process-environment (copy-sequence process-environment)))
      (when gptel-agent-runtime-ollama-models-directory
        (setenv "OLLAMA_MODELS"
                (expand-file-name gptel-agent-runtime-ollama-models-directory)))
      (start-process "ollama-serve" "*ollama-serve*"
                     gptel-agent-runtime-ollama-command "serve"))))

(defun gptel-agent-runtime-active-ollama-model ()
  "Return the first currently loaded Ollama model as a symbol, or nil.
This uses /api/ps, which reports loaded/running models. The function tolerates
both plist and vector shapes returned by `json-read'."
  (condition-case nil
      (let ((buf (url-retrieve-synchronously
                  (gptel-agent-runtime--ollama-url "/api/ps") t t 1)))
        (when buf
          (unwind-protect
              (with-current-buffer buf
                (goto-char (point-min))
                (when (re-search-forward "\n\n" nil t)
                  (let* ((json-object-type 'plist)
                         (json-array-type 'list)
                         (json-key-type 'keyword)
                         (data (json-read))
                         (models (plist-get data :models))
                         (first-model (car-safe models))
                         (name (plist-get first-model :name)))
                    (when (and (stringp name)
                               (not (string-empty-p name)))
                      (intern name)))))
            (kill-buffer buf))))
    (error nil)))
#+end_src

* Model-id normalization

gptel may return models as a bare symbol, a cons cell with metadata, or
a vector. =gptel-agent-runtime-model-id= picks the symbol id regardless of which
shape gptel hands back, so downstream code can compare model ids with
=eq=. =gptel-agent-runtime-backend-model-symbols= lists a backend's known models
through the same normalizer.

#+begin_src emacs-lisp
(defun gptel-agent-runtime-model-id (model)
  "Return the symbol model id from MODEL, tolerating gptel metadata forms."
  (cond
   ((symbolp model) model)
   ((and (consp model) (symbolp (car model))) (car model))
   ((and (vectorp model) (> (length model) 0) (symbolp (aref model 0)))
    (aref model 0))
   (t model)))

(defun gptel-agent-runtime-backend-model-symbols (backend)
  "Return BACKEND model ids as symbols, tolerating list/vector metadata."
  (when (and backend (fboundp 'gptel-backend-models))
    (mapcar #'gptel-agent-runtime-model-id
            (append (gptel-backend-models backend) nil))))
#+end_src

* Default local model selection

Selects the currently active Ollama model (or the configured fallback)
into =gptel-backend= / =gptel-model=, refreshes the directive for the
new runtime, and re-enables tools. Called interactively by the user,
automatically at package load (when an Ollama backend is registered),
and by the model router when it routes to a local profile.

#+begin_src emacs-lisp
;;;###autoload
(defun gptel-agent-runtime-use-default-local-model ()
  "Select the active or configured default local Ollama model when available."
  (interactive)
  (gptel-agent-runtime-start-ollama-if-needed)
  (when gptel-agent-runtime-ollama-backend
    (let ((model (or (and gptel-agent-runtime-prefer-active-ollama-model
                          (gptel-agent-runtime-active-ollama-model))
                     gptel-agent-runtime-default-local-model)))
      (unless (member model (gptel-agent-runtime-backend-model-symbols gptel-agent-runtime-ollama-backend))
        (when (fboundp 'gptel-backend-models)
          (setf (gptel-backend-models gptel-agent-runtime-ollama-backend)
                (append (gptel-backend-models gptel-agent-runtime-ollama-backend)
                        (list `(,model :capabilities (tool-use json)))))))
      (setq gptel-backend gptel-agent-runtime-ollama-backend
            gptel-model model)
      (when (fboundp 'gptel-agent-runtime-sync-directive-for-current-runtime)
        (gptel-agent-runtime-sync-directive-for-current-runtime))
      (when (fboundp 'gptel-agent-runtime-sync-tools)
        (gptel-agent-runtime-sync-tools))
      (message "gptel local model selected: %s%s"
               model
               (if (eq model gptel-agent-runtime-default-local-model)
                   (format " (%s)" gptel-agent-runtime-default-local-model-label)
                 " (active Ollama model)")))))
#+end_src

* Backend registry mutator

The setup file populates =gptel-agent-runtime-backends= directly with =setq=, but
the model router and =gptel-agent-runtime-select-model= use this helper to
register one entry at a time (replacing any existing entry with the same
display name).

#+begin_src emacs-lisp
(defun gptel-agent-runtime-register-model (name backend model)
  "Register NAME with BACKEND+MODEL in `gptel-agent-runtime-backends'.
Overwrites an existing entry with the same name."
  (setq gptel-agent-runtime-backends
        (cons (cons name (cons backend model))
              (cl-remove name gptel-agent-runtime-backends
                         :key #'car :test #'equal))))

;; Backwards-compat aliases for the legacy `my/gptel-*' names. Hosts may
;; bind these from setup scripts or from key bindings; the aliases keep
;; both names working until callers move to the new API.
(defalias 'my/gptel-model-id #'gptel-agent-runtime-model-id)
(defalias 'my/gptel-backend-model-symbols
  #'gptel-agent-runtime-backend-model-symbols)
(defalias 'my/gptel-register-model #'gptel-agent-runtime-register-model)
#+end_src

* Setup hand-off contract

The runtime package intentionally does NOT register any backends. The
consuming setup file is responsible for:

1. Requiring provider modules: =(require 'gptel-anthropic nil t)=,
   =(require 'gptel-openai nil t)=, =(require 'gptel-ollama nil t)=,
   =(require 'gptel-gemini nil t)=.
2. Calling =gptel-make-anthropic= / =gptel-make-openai= /
   =gptel-make-ollama= with the user's preferred endpoints, API keys,
   and model lists.
3. Populating =gptel-agent-runtime-backends= with the resulting
   =(DISPLAY-NAME . (BACKEND . MODEL))= entries (directly via =setq=
   or per-entry via =gptel-agent-runtime-register-model=).
4. Optionally setting =gptel-agent-runtime-ollama-backend= when an Ollama backend
   is registered, so =gptel-agent-runtime-use-default-local-model= and
   the mission-control buffer can find it.
5. Selecting an initial =gptel-backend= and =gptel-model=.

This split keeps the package model-neutral and lets each host carry its
own API keys, default model preference, and Ollama provisioning logic.
See the project README for a worked example of a host setup file.
