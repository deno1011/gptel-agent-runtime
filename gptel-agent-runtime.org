=package-vc-install= and, later, MELPA.

* Architecture Summary

=gptel-agent-runtime= is an Emacs-native agent runtime layer built on top
of [[https://github.com/karthink/gptel][gptel]]. gptel remains the transport, chat UI, streaming layer,
provider abstraction, and native tool interface. This package adds the
surrounding runtime: backend defaults, local-model behavior, prompts, web
helpers, response execution, workspace context, tool registration, and an
autonomous execution loop.

The current implementation is still monolithic, but the section boundaries
below are intentionally aligned with the future module split: core,
backends, media, web, executor, prompts, context, tools, and agent loop.

** Runtime Flow

#+begin_example
User prompt in gptel buffer
  -> gptel-send
  -> backend/model selection
  -> directive/system prompt sync
  -> optional chat router: direct chat or autonomous swarm session
  -> optional gptel tool schemas
  -> model response inserted into buffer
  -> gptel-post-response-functions
  -> response repair hooks
  -> Babel/AUTORUN/file-output execution
  -> org-display-inline-images
#+end_example

** Chat Router Startup

The chat router controls whether normal =gptel= sends stay as direct chat or
are intercepted and started as autonomous swarm sessions.

Package default:

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-chat-router-startup-mode 'off)
#+end_src

Useful personal configurations:

#+begin_src emacs-lisp :tangle no
;; Never auto-route normal chat. Use M-x gptel-agent-runtime-start manually.
(setq gptel-agent-runtime-chat-router-startup-mode 'off)

;; Ask before a complex prompt is routed to the swarm.
(setq gptel-agent-runtime-chat-router-startup-mode 'ask)

;; Automatically route prompts classified as complex.
(setq gptel-agent-runtime-chat-router-startup-mode 'auto)

;; Adjust how easily prompts become swarm tasks.
(setq gptel-agent-runtime-chat-router-min-score 3)
#+end_src

At runtime use =M-x gptel-agent-runtime-set-chat-router-startup-mode= to change
the startup preference. With a prefix argument, the command also saves the value
through Customize for future Emacs starts.

** Model Router Setup

The model router is separate from the swarm chat router. It chooses a registered
gptel backend/model before a normal send based on the current request, while the
chat router decides whether the request remains direct chat or becomes an
autonomous swarm session.

Package default:

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-model-router-enabled nil)
#+end_src

Enable it when you want model choice to follow the task:

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-model-router-enabled t)

;; Optional fallback when no specialist route matches.
(setq gptel-agent-runtime-model-router-default-profile 'local-balanced)
#+end_src

The router scores:

- complexity and required reasoning depth
- introspection, architecture, strategy, and self-improvement language
- context size
- code/tool/action risk
- web/current-fact needs
- privacy/local-only cues
- creativity, novelty, and open-ended exploration cues
- speed/cost cues such as "quick", "fast", or "cheap"

It then maps the request to a profile such as =local-fast=,
=local-balanced=, =local-reasoning=, =cloud-balanced=, =cloud-deep=,
=long-context=, or =cheap=. Profiles are matched against entries in
=my/gptel-backends= by display-name/model patterns. This is a routing heuristic,
not a live availability probe: it only knows which backends are registered in
Emacs, not whether a cloud key is valid or a local server is currently healthy.

Useful commands:

#+begin_src emacs-lisp :tangle no
M-x gptel-agent-runtime-model-router-preview
M-x gptel-agent-runtime-toggle-model-router
M-x gptel-agent-runtime-command-center   ;; press R for route preview
#+end_src

Manual model selection with =C-c M= remains available. Leave the router disabled
when you want a fixed model for a debugging session.

* Safety Layer

Risk levels, protected paths, and confirmation policy. Package defaults are open
for local testing and maximum functionality; harden them in personal or site
config when tool execution can modify important files or live Emacs state.

** Tool Policy Setup Examples

The package defaults are intentionally open for development and testing. The
default tool policy table adds trust/taint metadata, but it does not add extra
=:confirm= or =:default deny= rules. This keeps local models, raw-tool shims,
tests, and package experiments from being blocked by policy defaults.

Use =gptel-agent-runtime-tool-policy= for site-specific hardening. User policies
override =gptel-agent-runtime-default-tool-policy= per tool.

Example: ask before code, Elisp, writes, exports, and Org mutations.

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-require-confirmation-for-risky-actions t)

(setq gptel-agent-runtime-tool-policy
      '(("execute_code" . (:confirm always :taint untrusted))
        ("run_elisp" . (:confirm always :taint untrusted))
        ("org_export" . (:confirm write :taint trusted))
        ("write_file" . (:confirm write :taint trusted))
        ("write_org_file" . (:confirm write :taint trusted))
        ("add_todo" . (:confirm write :taint trusted))
        ("change_todo_state" . (:confirm write :taint trusted))
        ("set_deadline" . (:confirm write :taint trusted))
        ("add_tag" . (:confirm write :taint trusted))))
#+end_src

Example: deny shell/code execution completely, while still allowing other
tools according to their normal runtime checks.

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-tool-policy
      '(("execute_code" . (:default deny))
        ("run_elisp" . (:default deny))))
#+end_src

Example: allow write tools only under one project directory.

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-tool-policy
      '(("write_file" . (:confirm write
                         :paths ("~/projects/my-safe-project/")))
        ("write_org_file" . (:confirm write
                             :paths ("~/emacs-data/data/org/")))))
#+end_src

Example: restrict an especially powerful tool to the executor role.

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-tool-policy
      '(("execute_code" . (:confirm always
                          :agents ("executor")
                          :taint untrusted))))
#+end_src

Example: add command/code block patterns for a tool.

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-tool-policy
      '(("execute_code" . (:confirm always
                          :blocked-patterns ("\\bcurl\\b.*\\bsh\\b"
                                             "\\bchmod\\s-+-R\\b")
                          :taint untrusted))))
#+end_src

** Policy Presets

Use policy presets when you want a quick, inspectable safety posture without
hand-writing the policy alist each time.

#+begin_src emacs-lisp :tangle no
;; Maximum functionality for tests and local experiments.
(gptel-agent-runtime-apply-policy-preset 'open)

;; Daily-use default: ask before code, Elisp, writes, exports, and Org changes.
(gptel-agent-runtime-apply-policy-preset 'balanced)

;; Stronger mode: deny code/Elisp execution and ask before mutations.
(gptel-agent-runtime-apply-policy-preset 'strict)

;; Web/file reading only; no writes, exports, Org mutation, or code execution.
(gptel-agent-runtime-apply-policy-preset 'research-only)

;; Coding-oriented mode; allow code/write tools with confirmation, deny web fetches.
(gptel-agent-runtime-apply-policy-preset 'coding-only)

argument, the command saves the preset and the resulting policy variables
through Customize.

** Event Log Reliability

Runtime events are written to memory, the live swarm buffer, and optionally an
append-only file. File logging is useful for recovery and debugging, but it
must not break tests or live work when two Emacs processes append at the same
time.

The package therefore disables lockfile creation only for the event-log append
and ignores transient append errors by default:

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-event-log-ignore-write-errors t)
#+end_src

For strict debugging, make event-log write failures hard errors:

#+begin_src emacs-lisp :tangle no
(setq gptel-agent-runtime-event-log-ignore-write-errors nil)
#+end_src

  "Return list of all registered gptel-tool structs.

(defun my/gptel-select-model ()
  "Select backend+model — sets globally AND in the active gptel buffer."
  (interactive)
  (let* ((choice  (completing-read
                   (format "Model [current: %s]: " gptel-model)
                   my/gptel-backends nil t))

                      gptel-model   model)
